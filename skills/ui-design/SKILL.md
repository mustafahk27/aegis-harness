# UI Design

Use this skill to make interfaces clearer, calmer, and easier to use.

## Workflow

1. Identify the primary user goal and the single most important action.
2. Keep the product's existing visual language unless the user asks for a redesign.
3. Review hierarchy, spacing, alignment, typography, color, and component consistency.
4. Check responsive behavior and every important state: loading, empty, error, disabled, hover, focus, and success.
5. Tighten copy, labels, and micro-interactions so the UI feels deliberate.

## Checklist

- Make the main action obvious.
- Keep spacing and alignment consistent.
- Reuse existing components before inventing new ones.
- Ensure contrast, semantic structure, and keyboard accessibility.
- Adapt cleanly from mobile to desktop.
- Remove clutter, ambiguity, and duplicate visual weight.

## When editing UI code

- Prefer small, local changes.
- Suggest concrete layout, component, or copy changes instead of vague praise.
- Explain the tradeoff when proposing larger visual changes.
- Keep brand-breaking changes behind explicit approval.

## Good output

- A short critique of what works and what does not.
- A prioritized list of UI improvements.
- Concrete implementation notes for the next edit.
- A clear recommendation when the design direction is ambiguous.
