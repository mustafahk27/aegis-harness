# UI heuristics

## Hierarchy

- Put the primary action first and make it visually dominant.
- Use one clear headline, one supporting line, and one next step.
- Avoid giving secondary actions the same weight as the main CTA.

## Spacing and layout

- Prefer consistent spacing scales over ad hoc margins.
- Keep related items grouped and separate unrelated items with larger gaps.
- Align labels, inputs, and buttons to a shared grid.

## Typography and color

- Use type size and weight to show importance before using color.
- Reserve strong colors for actions, alerts, and status.
- Keep body text readable and avoid low-contrast gray-on-gray UI.

## Accessibility

- Preserve visible focus states.
- Make labels descriptive and avoid relying on placeholder text alone.
- Keep interactive targets large enough for touch and keyboard use.

## Common UI states

- Design loading, empty, error, and disabled states up front.
- Show what the user can do next when a screen has no data.
- Keep error copy specific and fix-oriented.

## Review questions

- What is the user trying to do here?
- What is the one action they should notice first?
- What is visually noisy or redundant?
- What would be hard to use on mobile or with a keyboard?
