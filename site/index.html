A coding harness for Pi

A disciplined way to let Pi touch your code.

It plans before changing code, prefers tests and debugging over guesswork, keeps diffs clean, and refuses risky actions.

01 — Behaviors

Plan-first
Pushes Pi to plan before changing code on anything non-trivial.
Test & debug
Prefers TDD and real debugging over guesswork.
Risk block
Blocks risky commands and secret-like values.
Modes
/mode feature, debug, refactor, review — or /mode to open a picker.
Explain
Surfaces clear block reasons plus /why and /explain.
Repo policy
Tune behavior per repo without touching the extension itself.

02 — Install

$ npm install deps
$ ./install.sh install
$ brew install gitleaks semgrep scan (optional)
$ pi start
$ /reload reload
